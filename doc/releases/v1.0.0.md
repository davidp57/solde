# Notes de version — v1.0.0 (2026-04-27)

## 🎉 Première version stable

Solde ⚖️ v1.0.0 est la première version de production. Elle consolide l'ensemble des fonctionnalités livrées depuis la v0.1 (2026-04-22) et marque la fin de la phase de recette.

---

## Nouveautés principales depuis la v0.8

### Notes de crédit (avoirs) — BIZ-129

Il est désormais possible de créer un avoir (note de crédit) directement depuis une facture client envoyée ou payée.

- Nouveau type de document `avoir` distinct des factures ordinaires
- Numérotation séparée automatique : `AV-2026-001`, `AV-2026-002`…
- Bouton « Créer un avoir » sur les factures envoyées et payées — les lignes sont pré-remplies avec les montants inversés
- Template PDF dédié avec en-tête **NOTE DE CRÉDIT**
- Le total négatif est autorisé et attendu pour les avoirs
- Traçabilité optionnelle : lien vers la facture d'origine (`credit_note_for_id`)

### Modèles d'e-mail configurables — BIZ-128

Le sujet et le corps des e-mails d'envoi de factures peuvent désormais être personnalisés depuis les paramètres SMTP.

- Variables disponibles : `{invoice_number}`, `{description}`, `{association_name}`, `{invoice_ref}`
- Laisser le champ vide conserve le comportement automatique par défaut
- Variables inconnues dans un modèle sont conservées telles quelles (pas d'erreur)

### Dialogue d'envoi de facture par e-mail — BIZ-127

Avant l'envoi, un dialogue de prévisualisation s'ouvre avec :
- Aperçu PDF de la facture intégré dans le dialogue
- Sujet et corps de l'e-mail éditables avant envoi
- Destinataire affiché (lecture seule)

### Chatbot IA et page Aide — BIZ-125, BIZ-126

- Sidebar de chat IA flottante (Google Gemini ou OpenAI), streaming en temps réel
- Page `/aide` avec manuel utilisateur intégré
- Configuration du fournisseur et de la clé API dans les paramètres (admin)
- Journalisation des questions dans la table `chat_log`
